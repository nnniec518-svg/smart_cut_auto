1素材扫描与净化

📁 VideoPurifier (processor.py)

对原始视频素材进行预处理，提取有效片段并分类。

处理流程

步骤说明
1. 音频提取使用 FFmpeg 提取视频音频
2. VAD 检测使用 FunASR 进行语音活动检测，获取静音段
3. 音量分析计算平均音量，低于阈值(-40dB)标记为 B_ROLL
4. ASR 识别调用 FunASR 获取词级时间戳和文本
5. 提示词检测正则匹配 "321"、"走"、"开始" 等开拍提示词
6. 逻辑过滤过滤无效片段（过短、过密、置信度低）
7. 质量评估计算 A_Roll 分数，入围 B_Roll 候选

输出

  • track_type: "A_ROLL" 或 "B_ROLL"
  • valid_start_offset: 有效片段起始时间（跳过提示词）
  • asr_text: 完整 ASR 文本
  • segments: 带时间戳的文本片段列表

2文案语义解析

🧠 Matcher (matcher.py)

将目标文案与素材进行语义匹配。

核心功能

功能说明
语义模型使用 paraphrase-multilingual-MiniLM-L12-v2 多语言模型
文本预处理去除空格、标点，统一大小写
相似度计算余弦相似度 + 词重叠 + LCS 综合评分
分句处理按句号、问号、感叹号分句

算法优化 (v2.0)

🔤 语义指纹去重

使用 Levenshtein 编辑距离 (< 5) 去除重复文案

📍 滑动窗口匹配

只搜索最近30个句子，强制按顺序匹配

⚖️ 索引距离惩罚

离预期位置越远扣分越多，强制顺序匹配

⏱️ 时间戳Padding

前后各加0.2秒，防止ASR截断吞字

3序列规划与匹配

📝 Assembler / SequencePlanner

根据目标文案，从素材库中选择最佳片段组成序列。

核心算法

阶段说明
1. 全文对齐脚本驱动的全局匹配
2. 语义归一化去除标点、助词，提取核心词
3. 唯一性锁定每个素材片段只能使用一次
4. 末次优先同内容多版本时，选择最后录制的
5. 排序优化完整性分数 + 录制时间综合排序

缓存机制

logic_version 配置用于强制刷新缓存：

  • 修改 config.yaml 中的 logic_version
  • 系统自动检测版本变化，清理旧缓存

4FFmpeg 渲染合成

🎥 VideoAutoCutter / VideoProcessor

使用 FFmpeg 进行视频裁剪、拼接、转码。

处理流程

步骤说明
1. 时间裁剪根据时间戳提取有效片段
2. 分辨率统一统一缩放到 1080x1920 (9:16竖版)
3. 帧率转换统一转换为 30fps
4. 视频拼接使用 concat 滤镜合并多个片段
5. 音频归一化loudnorm 滤镜统一音量

输出配置

  • 分辨率: 1080 x 1920
  • 帧率: 30fps
  • 编码: libx264, CRF=18
  • 音频: AAC, 128kbps

命令行用法

# 基本用法

python main.py -s script.txt -c my_config.yaml

# 强制使用CPU
python main.py --cpu

  • temp/ 目录下的临时文件（保留最终输出）
  • storage/materials.db 数据库文件
  • logs/ 目录下的日志文件
