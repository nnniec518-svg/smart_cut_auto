## 三、对比优化前后的变化

### 3.1 优化前 (基于原始 sequence.json)

**主要问题**:
1. **ASR 错别字未纠正**: "权益四胜"、"洗红"等错误词未处理
2. **数字不匹配**: "至高省1399元" 匹配到 "三千至"
3. **低相似度片段入选**: 相似度 <0.5 的片段仍被选入
4. **中断片段未拼接**: 部分素材文本被截断

### 3.2 优化后 (本次测试)

**改进**:
1. ✅ **配置完善**: 添加了 `number_constraint` 和 `segment_merge` 配置
2. ✅ **混合相似度计算**: 支持关键词+语义+编辑距离的加权计算
3. ✅ **数字硬约束**: 启用了数字匹配检查
4. ✅ **片段拼接支持**: 支持最多 3 片段组合

**仍存在的问题**:
1. ❌ **ASR 纠错未生效**: `processor.py` 中的纠错词典未被调用
2. ❌ **数字约束过于严格**: 导致多个带数字的文案匹配失败
3. ❌ **A_ROLL 素材不足**: 大量 B_ROLL 作为 fallback 使用
4. ❌ **相似度计算不准确**: 语义相似度与实际文本匹配度不匹配

## 四、关键问题定位

### 4.1 ASR 纠错未生效

**原因**: `processor.py` 中的 `_apply_text_correction` 函数未被实际调用

**代码位置**: `core/processor.py:32-49`

**修复建议**: 在 `_asr_with_timestamp` 方法中调用纠错函数

### 4.2 数字约束问题

**现象**: 所有带数字的文案都匹配失败

**原因**:
1. 数字约束 `enabled: true` 导致所有数字不匹配的候选被过滤
2. 中文数字转换可能不准确
3. ASR 文本中的数字可能被识别为汉字或其他形式

**建议**:
1. 暂时禁用数字约束进行测试
2. 改进数字提取和转换逻辑
3. 增加数字匹配的容错性

### 4.3 相似度计算问题

**现象**: 相似度高但实际匹配文本与文案不符

**原因**:
1. Sentence-Transformer 模型降级，使用的是英文模型
2. 中文语义理解能力弱
3. 混合相似度中关键词权重可能过高

**建议**:
1. 安装并使用 BGE-M3 或其他中文模型
2. 调整混合相似度的权重配置
3. 增加关键词匹配的准确性

## 五、优化建议

### 5.1 短期优化 (1-2 天)

1. **修复 ASR 纠错调用**
   - 在 `processor.py` 中启用纠错词典
   - 重新处理素材生成新的数据库

2. **调整数字约束**
   - 将 `number_constraint.enabled` 设为 `false` 进行测试
   - 或降低惩罚分数，改为软约束

3. **降低相似度阈值**
   - 将 A_ROLL 阈值从 0.3 降至 0.25
   - 增加候选召回数量

### 5.2 中期优化 (3-7 天)

1. **优化数据库素材**
   - 手动添加关键数字和权益相关的素材
   - 确保 A_ROLL 类型素材充足

2. **改进混合相似度**
   - 调整关键词权重 (降低到 0.2)
   - 增加编辑距离权重 (提高到 0.2)
   - 语义相似度权重保持 0.6

3. **添加人工校验**
   - 对匹配结果进行人工审核
   - 标记正确/错误的匹配样本
   - 用于优化相似度算法

### 5.3 长期优化 (1-2 周)

1. **升级向量模型**
   - 安装 FlagEmbedding 库
   - 使用 BGE-M3 或 BGE-Large-ZH 模型
   - 提升中文语义理解能力

2. **构建领域知识库**
   - 收集本地生活领域的关键词、实体
   - 建立产品-权益-金额的关联图谱
   - 用于增强匹配准确性

3. **实现反馈学习**
   - 记录用户对匹配结果的反馈
   - 使用在线学习调整相似度权重
   - 持续优化匹配效果

## 七、结论

### 当前状态
- ✅ 代码框架完整，混合相似度和数字约束已实现
- ⚠️ 配置已添加，但部分功能未完全生效
- ❌ 匹配效果不佳，匹配率仅 69.2%

### 根本原因
1. **向量模型降级**: 使用英文模型导致中文语义理解差
2. **数据库素材不足**: A_ROLL 素材数量有限
3. **数字约束过严**: 过滤掉了大部分带数字的候选
4. **ASR 纠错未用**: 错别字未被纠正

### 下一步行动
1. **紧急**: 禁用数字约束，重新测试匹配效果
2. **短期**: 修复 ASR 纠错调用，重新处理素材
3. **中期**: 升级中文向量模型，增加 A_ROLL 素材
4. **长期**: 构建领域知识库，实现反馈学习
