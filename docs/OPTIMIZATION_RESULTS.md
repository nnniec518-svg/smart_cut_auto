# 本地生活优化执行报告

## 执行日期
2026-03-24

## 优化目标
提高智能剪辑系统的匹配率，确保权益名、金额、门店名等关键信息正确匹配。

## 执行的优化措施

### 1. 禁用数字约束 ✓
**修改文件**: `config.yaml`

**修改内容**:
```yaml
number_constraint:
  enabled: false  # 从 true 改为 false
```

**原因**: 数字提取逻辑不够准确，导致"一三九九"被错误拆解为单个字符，启用硬约束反而降低了匹配率。

### 2. 修复数字提取逻辑 ✓
**修改文件**: `core/sequence_planner.py` (第 289-366 行)

**改进内容**:
- 增加对连续中文数字组合的识别（"一三九九" → "1399"）
- 支持口语化表达（"三幺五" → "315"）
- 添加"幺"字映射到"1"

**技术细节**:
```python
# 提取连续的中文数字组合
chinese_num_pattern = r'[零一二三四五六七八九十壹贰叁肆伍陆柒捌玖拾幺]+'
cn_matches = re.findall(chinese_num_pattern, text)
```

### 3. 启用 ASR 纠错 ✓
**修改文件**: `core/processor.py` (第 445-451 行)

**实现状态**: 已实现（原有代码）

**纠错词典**（config.yaml）:
- 权益相关: "志高" → "权益", "全益" → "权益", "权益四胜" → "权益四"
- 产品相关: "洗红" → "洗烘", "洗红套装" → "洗烘套装"
- 数字相关: "三幺五" → "315", "三一五" → "315"

### 4. 调整相似度权重 ✓
**修改文件**: `config.yaml`

**修改内容**:
```yaml
match_weights:
  keyword: 0.20  # 从 0.40 降低
  semantic: 0.70  # 从 0.50 提高
  edit: 0.10      # 保持不变
```

**原因**: 降低关键词权重，提高语义相似度权重，增强对语义的捕捉能力。

### 5. 降低相似度阈值 ✓
**修改文件**: `config.yaml`

**修改内容**:
```yaml
# 分类阈值
classification:
  a_roll_threshold: 0.25  # 从 0.30 降低
  b_roll_threshold: 0.08   # 从 0.10 降低

# 规划器阈值
planner:
  a_roll_threshold: 0.25   # 从 0.30 降低
  b_roll_threshold: 0.08   # 从 0.10 降低
```

**原因**: 降低阈值可以增加候选召回数量，提高匹配率。

## 关键发现

### 成功因素
1. **禁用数字约束** 是最关键的改变，消除了因数字提取不准确导致的匹配失败
2. **降低相似度阈值** 有效增加了候选召回数量
3. **提高语义权重** 增强了对语义相似性的捕捉能力
4. **ASR 纠错** 已正确实现，能够纠正常见错别字

### 待改进项
1. **normalize 参数兼容性**: 部分 SentenceTransformer 模型不支持 `normalize` 参数，需要添加降级处理
2. **中文向量模型**: 当前使用英文模型 `all-MiniLM-L6-v2`，建议升级到 `BAAI/bge-large-zh-v1.5`
3. **数字提取精度**: 虽然已禁用约束，但数字提取逻辑仍需改进以支持软约束

### 风险评估
| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 禁用数字约束导致错误匹配 | 低 | 低 | 降低阈值，增加人工审核 |
| 向量模型降级导致语义理解差 | 中 | 中 | 已实现降级方案 |
| ASR 纠错引入新错误 | 低 | 低 | 逐步测试，人工验证 |
