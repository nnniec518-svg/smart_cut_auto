# 本地生活优化实施步骤

## 紧急修复 (立即执行)

### 1. 禁用数字约束
**目的**: 提高匹配率，避免因数字提取不准确导致的匹配失败

**修改**: `config.yaml`
```yaml
number_constraint:
  enabled: false  # 暂时禁用
```

### 2. 修复数字提取逻辑
**目的**: 正确提取中文数字组合

**修改**: `core/sequence_planner.py:286-328`

**当前问题**: "一三九九" 被拆解为 `{'1', '3', '9'}` 而非 `{'1399'}`

**修复方案**: 改进 `_extract_numbers` 方法，添加中文数字组合识别

### 3. 启用 ASR 纠错
**目的**: 纠正 ASR 识别的错别字

**修改**: `core/processor.py`

**需要添加**: 在 `_asr_with_timestamp` 方法中调用 `_apply_text_correction`

## 短期优化 (1-2 天)

### 4. 调整相似度权重
**目的**: 降低关键词权重，提高语义相似度权重

**修改**: `config.yaml`
```yaml
match_weights:
  keyword: 0.20  # 从 0.40 降低
  semantic: 0.70  # 从 0.50 提高
  edit: 0.10      # 保持不变
```

### 5. 降低相似度阈值
**目的**: 增加候选召回数量

**修改**: `config.yaml`
```yaml
planner:
  a_roll_threshold: 0.25  # 从 0.30 降低
  b_roll_threshold: 0.08  # 从 0.10 降低
```

### 6. 重新处理素材
**目的**: 应用 ASR 纠错后重新生成数据库

**步骤**:
1. 清空或备份数据库
2. 重新运行素材处理流程
3. 验证纠错效果

## 中期优化 (3-7 天)

**目的**: 提升中文语义理解能力

**安装**:
```bash
pip install FlagEmbedding
```

**修改**: `config.yaml`
```yaml
models:
  embedding_model: "BAAI/bge-large-zh-v1.5"
```

### 8. 改进中文数字提取

## 长期优化 (1-2 周)

### 10. 实现软数字约束
**目的**: 保留数字匹配的重要性，但不完全排除候选

**方案**:
- 数字匹配：分数不变
- 数字不匹配：分数减半或降低 0.3
- 保留候选供后续选择

### 11. 构建训练数据集
**目的**: 使用人工标注数据优化相似度权重

**步骤**:
1. 收集匹配结果（正确/错误）
2. 标注每个匹配的质量评分
3. 使用回归分析优化权重配置

### 12. 实现反馈学习
**目的**: 根据用户反馈持续优化

**方案**:
- 记录用户接受/拒绝的匹配
- 使用在线学习调整权重
- 定期更新模型参数

## 测试计划

### 阶段 1: 紧急修复验证
- 禁用数字约束后重新运行匹配
- 验证匹配率提升至 85% 以上
- 检查关键文案（权益、金额）的匹配效果

### 阶段 2: 功能验证
- 验证 ASR 纠错是否生效
- 检查"洗红"是否纠正为"洗烘"
- 验证相似度权重调整后的效果

### 阶段 3: 全面测试
- 升级向量模型后重新测试
- 对比优化前后的匹配质量
- 收集用户反馈

### 阶段 4: 生产部署
- 在测试环境验证通过后部署
- 监控匹配效果和性能
- 持续优化和调整
